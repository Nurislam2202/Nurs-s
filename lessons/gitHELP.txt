*команды которые пишутся тоько один раз, в начале проекта
git init - инициализирует проект(.git)
git remote add origin 'ссылка на ваш репозиторий' - привязывает репозиторий и ваш проект


file: .gitignore - написать сюда файлы, которых гит должен игнорить


*команды которые пишутся каждый раз, когда мы хотим отправить что-то в репозиторий
git add namefile\. - добаляет в репозиторий ваши файлы
git commit -m 'lesson1' - чекпоинт, сохронение
git push origin master\main - отправка в ваш репозиторий


*полезные команды
git status -s - показывает этапы файлов
git rm --cached namefile -r - рип файлов в репозитории

git clone "url ссылка" - скачивает проект из гит репозитория
git remote remove origin - отвязывает проект от репозитория

git branch - проверка веток
git branch name - создание веток
git checkout name - перепрыгивание на ветку
git merge - сливает обновления из ветки, которую вы указави в ветку, в которой вы находитесь


*команды для виртуальной среды
deactivate - деактивирует
(указать полный путь виртуальной среды) activate - для активации, пример: (.venv\Scripts\activate)
python -m venv namefile - установка виртуальной среды, с тем названием которой вы указали, вместо namefile
pip install name - (pip - пакетный менеджер питона), скачивание модулей, вместо name указывете модуль,
определенная версия скачивается так, пример: colorama==0.4.6
pip freeze - проверка модулей
pip freeze > namefile (обычно это requirements.txt) - создание файла, и прописание туда названия всех имеющихся модулей
pip install -r namefile - скачивание модулей с файла (-r - от слово read)
